# Security Policy

## Supported versions

Security fixes are applied to the latest released version and the current `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub private vulnerability reporting when enabled, or contact the repository owner privately through GitHub.

Include:

- affected version or commit;
- reproduction steps or proof of concept;
- expected and observed behavior;
- impact assessment;
- suggested mitigation when available.

## Scope

Relevant reports include bypasses, denial-of-service risks, unsafe default behavior, dependency vulnerabilities, data exposure, and adapter behavior that could cause unintended moderation actions.

The library reports incidents but does not execute destructive actions by default. Integrators remain responsible for permissions, rate limits, evidence retention, and safe action handlers.
